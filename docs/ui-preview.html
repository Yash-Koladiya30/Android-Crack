REST API

FastAPI behind android-crack serve. Every call lands in the audit log.
GET/health — liveness probe
GET/devices — list ADB devices
GET/devices/{serial}/info — model / Android version / SDK / SoC
GET/devices/{serial}/battery — dumpsys battery
GET/devices/{serial}/apps?kind=third_party — package list
POST/devices/{serial}/screenshot — capture + pull
GET/devices/{serial}/wifi/saved — saved SSIDs (no passwords)
GET/devices/{serial}/root — root / Magisk heuristics
POST/devices/{serial}/exports/sms — dump SMS
POST/devices/{serial}/exports/contacts — dump contacts
POST/devices/{serial}/exports/calls — dump call log
GET/audit/tail?limit=N — recent audit events

Capability Surface

~70 commands grouped by domain.

connect

tcp + Android 11 pair + disconnect + LAN scan

shell

interactive + one-shot, keycodes + text input

apps

list, install (+ split), uninstall, launch, force-stop, clear, restart, perm grant/revoke, extract APK

files

ls / pull / push + bucket pulls (WhatsApp, Camera, Screenshots, Downloads, Music, Movies)

export

SMS, contacts, call log, logcat

media

screenshot, screen-record, scrcpy mirror, mic / device audio stream + record

comms

send SMS, open URL on device

keycodes

interactive REPL: 19 named codes + arbitrary keycode + text input

network

ip / route / DNS, ping, forward / reverse, LAN scan

wifi

status, ip, enable / disable, saved SSIDs (no passwords)

diag

battery, locale, root heuristics, dev-settings, stay-on

exploit

msfvenom payload + msfconsole handler + opt-in verifier toggle

cluster

parallel info / screenshot / shell across N devices

audit

SQLite trail of every command

tui

Textual app with device table + log pane

serve

FastAPI REST mode for automation
